Investigación sobre Seguridad de Páginas Web y Frameworks

OWASP (Open Web Application Security Project)

OWASP es una organización sin fines de lucro enfocada en mejorar la seguridad del software. Proporciona una serie de recursos, incluyendo documentación, herramientas y estándares que ayudan a las organizaciones a desarrollar, mantener y operar aplicaciones de manera segura.

Principales Recursos y Proyectos de OWASP:

  • OWASP Top Ten: Una lista de los diez riesgos más críticos de seguridad en aplicaciones web. Esta lista se actualiza periódicamente y es una referencia fundamental para los desarrolladores.
  • OWASP ASVS (Application Security Verification Standard): Un marco de requisitos de seguridad que se puede utilizar para diseñar, desarrollar y probar aplicaciones web.
  • OWASP ZAP (Zed Attack Proxy): Una herramienta de pruebas de seguridad de aplicaciones web que puede ser utilizada para encontrar vulnerabilidades en las aplicaciones.
  • OWASP SAMM (Software Assurance Maturity Model): Un modelo que ayuda a las organizaciones a formular e implementar una estrategia de seguridad de software a medida.

Principales Riesgos de OWASP Top Ten:

  • Inyección
  • Autenticación rota
  • Exposición de datos sensibles
  • Entidades externas XML (XXE)
  • Control de acceso roto
  • Configuración de seguridad incorrecta
  • Secuencias de comandos entre sitios (XSS)
  • Deserialización insegura
  • Usos de componentes con vulnerabilidades conocidas
  • Registro y monitoreo insuficientes

SSL / TLS (Secure Sockets Layer / Transport Layer Security)

SSL y TLS son protocolos criptográficos diseñados para proporcionar comunicaciones seguras a través de una red. Aunque SSL es el antecesor de TLS, ambos términos a menudo se usan indistintamente para referirse a la seguridad de las comunicaciones en internet.

Características y Funciones de SSL/TLS:

  • Cifrado: Proporciona confidencialidad mediante el cifrado de los datos transmitidos.
  • Autenticación: Verifica la identidad de los participantes en la comunicación.
  • Integridad de Datos: Asegura que los datos no hayan sido alterados durante la transmisión.
  • Certificados Digitales: Utilizados para verificar la identidad de los servidores y en algunos casos, de los clientes.

Proceso de Establecimiento de una Conexión SSL/TLS:

  • Handshake: Proceso de negociación inicial donde se establece el protocolo de cifrado y se intercambian certificados.
  • Cifrado de Datos: Una vez establecido el canal seguro, se procede al cifrado de los datos transmitidos.
  • Terminación de la Conexión: Finalización de la comunicación segura.

Frameworks

React

React es una biblioteca de JavaScript desarrollada por Facebook para la construcción de interfaces de usuario. Es conocida por su rendimiento y flexibilidad.

Características Principales de React:

  • Componentes: React está basado en componentes reutilizables.
  • Virtual DOM: Mejora el rendimiento al actualizar solo los cambios en el DOM.
  • Unidirectional Data Flow: Facilita la gestión de datos predecible.
  • JSX: Sintaxis que combina JavaScript y HTML, simplificando la creación de componentes.

Angular

Angular es un framework de desarrollo web creado por Google. Utiliza TypeScript y es ideal para aplicaciones de gran escala.

Características Principales de Angular:

  • MVC (Model-View-Controller): Facilita la separación de la lógica de la aplicación y la interfaz de usuario.
  • Two-Way Data Binding: Sincroniza automáticamente el modelo y la vista.
  • Dependency Injection: Gestión eficiente de dependencias, facilitando la modularidad y la reutilización del código.
  • TypeScript: Un lenguaje tipado que ayuda a prevenir errores y mejora la mantenibilidad del código.

Vue.js

Vue.js es un framework progresivo de JavaScript para construir interfaces de usuario. Es conocido por su simplicidad y facilidad de integración.

Características Principales de Vue.js:

  • Reactividad: Sistema de reactividad eficiente para gestionar cambios en los datos.
  • Componentes: Estructura basada en componentes reutilizables.
  • Directivas: Facilitan la manipulación del DOM de manera declarativa.
  • Ecosistema Modular: Integración con bibliotecas y herramientas para construir aplicaciones completas.
